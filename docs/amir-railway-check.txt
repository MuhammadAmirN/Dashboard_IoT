CHECKLIST

1. [x] Mengecek konfigurasi Railway agar aplikasi tetap berjalan setelah deploy.
2. [x] Memastikan environment variable database sudah benar.
3. [x] Mengecek endpoint API production bisa menerima data sensor.

Yang sudah dicek dan disesuaikan:
1. URL asset dipaksa memakai HTTPS saat production supaya CSS tidak diblokir browser.
2. Railway build command disesuaikan agar lebih stabil di Railpack.
3. Start command disiapkan agar aplikasi berjalan dengan port yang dipakai Railway.
4. Database production diarahkan ke MySQL Railway agar data sensor tidak hilang saat redeploy.
5. Environment variable database untuk service WebIoT sudah mengarah ke MySQL Railway:
   - DB_CONNECTION=mysql
   - DB_HOST=mysql.railway.internal
   - DB_PORT=3306
   - DB_DATABASE=railway
   - DB_USERNAME=root

File utama yang terkait:
- app/Providers/AppServiceProvider.php
- railway.json

Branch yang dipakai:
git checkout -b amir_endpoint_api

Catatan:
Semua perubahan untuk tugas Amir sebaiknya dikerjakan di branch ini, lalu di-push ke GitHub sebelum merge ke main.

Pemberitahuan ke tim:
Konfigurasi environment variable database di Railway sudah dicek dan sudah benar. Silakan lanjut kerja di branch masing-masing, jangan push ke main.
