ATURAN WAJIB SEBELUM MULAI

1. Ambil update terbaru dari branch utama.

   git pull origin main

2. Buat branch sendiri sesuai nama dan tugas.

   git checkout -b fitur/nama-tugas-nama

3. Jangan push langsung ke main.

4. Setelah selesai, commit dan push branch masing-masing.

   git add .
   git commit -m "menambahkan tugas nama"
   git push origin fitur/nama-tugas-nama

5. Setelah push, buat Pull Request atau minta ketua project merge ke main.

ALUR KERJA SETIAP ANGGOTA

1. Buka terminal di folder project.

   cd C:\.s6peminatan\webiot

2. Ambil update terbaru.

   git pull origin main

3. Buat branch sesuai tugas.

   git checkout -b fitur/nama-tugas-nama

4. Kerjakan file sesuai jobdesk.

5. Cek perubahan.

CATATAN PENTING

Jika ada conflict saat pull atau merge, jangan asal hapus kode.
Tanyakan dulu ke anggota yang mengerjakan file tersebut.

Jangan mengubah file di luar jobdesk kecuali sudah disepakati.
Tujuannya agar Git graph rapi dan perubahan setiap orang terlihat jelas.
